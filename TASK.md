# Контекст задачи

Команды контент-менеджеров и маркетологов работают с огромным потоком информации: статьями, анонсами, видео, постами в соцсетях. Управлять этим объемом и оценивать эффективность публикаций — сложная аналитическая задача.

Большинство команд используют простые таблицы (например, Google Sheets, Excel, соцсети, CMS, аналитические панели), но такие инструменты:

- Не умеют собирать данные автоматически из разных источников;
- Не анализируют тексты комментариев;
- Не позволяют строить прогнозы или находить закономерности.

Отсутствует интегрированное решение, которое в едином пространстве агрегирует статистику и помогает принимать стратегические решения на основе данных.

Мы предлагаем вам создать такое интегрированное решение на MWS Tables. Это платформа для табличного взаимодействия данных внутри экосистемы MWS.

## Постановка задачи

Разработайте умный реестр контента в MWS Tables, который:

- Подключается к внешним источникам данных.
- Автоматически собирает статистику по материалам, размещенным на этом ресурсе, в пространство вашего реестра.
- Визуализирует данные.

### Базовая функциональность умного реестра контента:

- Агрегация данных по материалам из открытых API.
- Автоматическое обновление таблиц в MWS Tables.
- Отображение статистики (например, просмотров, лайков, комментариев).
- Отчет по контенту с визуализацией (графики, сортировка, фильтры).
- Возможность экспортировать данные.

### Задачи со звездочкой:

- Ваше решение в MWS Tables сопряжено с LLM, которая:
  - Анализирует тексты комментариев (тональность, ключевые темы, настроение аудитории).
  - Строит прогноз потенциальной популярности новых материалов.
  - Имеет отдельный веб-интерфейс чат-бота, который отвечает на вопросы по реестру контента на естественном языке. Например: «Какой был самый популярный пост на прошлой неделе?»

### Фичи со звездочкой:

- Подключение LLM для анализа комментариев и тональности.
- Прогнозирование потенциальной популярности материалов (через LLM).
- Интеграция чат-бота, который отвечает на вопросы:
  - «Какой лонгрид был самым популярным на прошлой неделе?»
  - «Почему этот пост вызвал столько негативных комментариев?»
- Возможность предложить пользователю улучшения контент-плана (AI-рекомендации).

## Детализация технических требований к решению

### 1. Как это должно работать

- Пользователь открывает свое пространство в MWS Tables.
- Система собирает данные о контенте по API и обновляет таблицу.
- При клике на материал можно увидеть детали и прогноз вовлеченности.
- Пользователь может посмотреть визуальное отображение статистики.
- **Со звездочкой.** Пользователь может задать вопрос через разработанный вами веб-интерфейс чат-бота. Например: «Какие посты получили больше всего лайков?» — бот возвращает ответ и объяснение.

### 2. Что по ограничениям

- Обязательно использование MWS Tables.
- Обязательно подключение к внешним источникам или источнику данных по API.
- Модель обучать не требуется: используется готовая LLM через API.
- Допускается применение любых open-source-библиотек и API, доступных без лицензий.

### 3. На что обращаем внимание

- На интуитивность взаимодействия пользователя с пространством.
- Работающую интеграцию с реальными данными.
- Качество визуализации и юзабилити.
- Наличие предиктивного элемента через LLM.

### 4. В каком виде представить результат

- Доступ к пространству в MWS Tables с вашим решением.
- Если вы разработали чат-бота:
  - Разверните решение на удаленном сервере и предоставьте проверяющим ссылку для доступа. Также предоставьте открытый доступ к репозиторию GitHub, в котором находится ваше решение.

## Подробнее о MWS Tables

**MWS Tables** ([https://mws.ru/dev-tools/mws-tables/](https://mws.ru/dev-tools/mws-tables/))— no-code-платформа для автоматизации рабочих процессов и управления данными. Сервис позволяет создавать приложения на базе таблиц без привлечения разработчиков и содержит популярные инструменты для решения задач в разных сферах — от маркетинга и HR до управления продуктом.

Ссылка на документацию: [https://support.mts.ru/mws_tables#rabota-s-prostranstvami](https://support.mts.ru/mws_tables#rabota-s-prostranstvami)

Очень рекомендуем пройти обучение после регистрации, чтобы ознакомиться с основными возможностями и инструментами сервиса.

### Совместная работа

Платформа поддерживает командную работу: роли и доступы настраиваются для каждого пользователя. Можно ограничить редактирование, открыть таблицу только для просмотра или распределить доступы между участниками в зависимости от задач.

### Автоматизации

В системе предусмотрены автоматические сценарии, которые выполняют заданные действия при наступлении определенных условий. Можно настроить обновление данных, отправку уведомлений и другие базовые процессы без использования кода.

### Формулы

MWS Tables поддерживает формулы, которые позволяют выполнять вычисления внутри таблиц. Формулы используются для расчетов, преобразования данных и формирования вспомогательных полей. Настройка и изменение выполняются напрямую в интерфейсе.

### Дашборды

Платформа позволяет создавать дашборды, где данные визуализируются в виде графиков, диаграмм и других элементов. Дашборды обновляются автоматически и помогают анализировать показатели, отслеживать динамику и представлять ключевую информацию в удобной форме.

### Интеграции

Платформа имеет API для обмена данными с внешними сервисами. Через API можно получать, изменять и добавлять записи. Интеграции используются для автоматического взаимодействия MWS Tables с другими приложениями и системами.

## Критерии оценки решения на хакатоне

Решение участников оценивается по четырем критериям. По каждому из них можно набрать от 1 до 5 баллов. Каждый критерий имеет свой вес в процентах от итогового балла за данный критерий:

- **60%** — решение включает рабочий прототип умного реестра в MWS Tables. Соответствие техническим требованиям и функциональности.
- **25%** — качественно представленная концепция решения с обоснованным выбором подходов и архитектуры пространства в MWS Tables.
- **15%** — качество логики и UX-решения

**Дополнительные баллы:**

- **+5 баллов** — реализация чат-бота (frontend + backend)
- **+1 балл** — дополнительные необходимые функции для решения бизнес-кейса, которые не прописаны в рамках ТЗ.
- **+1 балл** — качество защиты и четкое донесение преимуществ своего решения перед жюри.

## Приложение. Откуда взять LLM и как ее подключить

Для реализации задач со звездочкой вам требуется подключить готовую языковую модель (LLM) через API. Обучать модель самостоятельно не нужно — достаточно использовать один из доступных сервисов, предоставляющих доступ к LLM по HTTP-запросам.

Мы рекомендуем использовать [openrouter.ai/](https://openrouter.ai/) — агрегатор моделей, предоставляющий единый API для разных LLM.

После регистрации вы должны создать единый API-ключ, который можете использовать для запросов к разным моделям. Документацию вы можете найти по этой ссылке: [https://openrouter.ai/docs/quickstart](https://openrouter.ai/docs/quickstart)
